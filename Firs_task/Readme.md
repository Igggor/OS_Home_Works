# Первое задание по Операционным Системам.
## №1
### Условие задачи:
Напишите программу useless (Unix System Extremely Late Execution
Software System), которая читает файл и запускает указанные в нём
программы с указанной задержкой от времени старта самой программы
useless. Формат записи в файле: «время задержки в секундах»
«программа для выполнения».
Например:
```shell
5 ls
1 ./test
3 ps
```

### Моя реализация:
У нас есть файл, например commands.txt:

```txt
5 ls
1 ./test
3 ps
```

Компилирую программу:
```bash
gcc useless.c -o useless
```

И запускаю:
```bash
./useless commands.txt
```

## №2

Для того чтобы не допустить потерю информации при сбое диска,
обычно используют резервное копирование файлов (backup).
Простейшей формой backup'а является копирование файлов из одного
каталога в другой. Этот способ требует много времени и места на диске.
Напишите программу, осуществляющую более интеллектуальный
подход. Программа должна брать из командной строки два параметра:
имена исходного каталога и каталога назначения. Она должна
рекурсивно сканировать исходный каталог, делать копии всех файлов,
для которых ранее не делались копии или которые были изменены с
момента последнего backup'а, размещая их в соответствующих местах
директории назначения. После копирования для каждого файла должна
вызываться команда сжатия gzip. Это уменьшит требуемое дисковое
пространство, а файл будет переименован с добавлением
расширения .gz. Все возникающие ошибки (нет исходного каталога,
файл не доступен для чтения и т.д.) должны корректно обрабатываться с
выдачей соответствующего сообщения.


### Моя реализация:
Компиляция программы:
```bash
gcc backup.c -o backup
```

И запуск резервного копирования:
```bash
./backup /путь/к/исходному /путь/к/резервному
```

Файлы будут копироваться и сжиматься (gzip), создавая .gz версии только если они новые или изменённые.
